# Statistical Analysis of Student Academic Performance

This project was prepared for **IT2022E - Applied Statistics and Experimental Design**.  
It uses the **UCI Student Performance** dataset for the Mathematics course, with 395 student
records.

## 1. Objective

The project demonstrates a complete applied statistics workflow in Python:

1. Prepare the dataset and produce descriptive statistics.
2. Classify variables using measurement theory.
3. Estimate confidence intervals and run hypothesis tests.
4. Analyze correlation and linear regression models.
5. Propose, size, and simulate an experimental design.
6. Discuss error sources, limitations, and how the results should be interpreted.

Important note: the original dataset is observational. Therefore, the EDA, hypothesis testing,
and regression results describe **associations**, not causal effects. The experimental design
section proposes how a future randomized study could address a causal question.

## 7. Data and Analysis Conventions

- Original file: `data/raw/student-mat.csv`
- Main working file: `data/processed/student_mat_clean.csv`
- Outcome variable: `G3`, score range 0-20
- Significance level: `ALPHA = 0.05`
- Reproducibility seed: `RANDOM_SEED = 42`
- Keep observations with `G3 = 0` in the main analysis
- Do not remove or winsorize `absences` in the main dataset
- For multiple testing, conclusions use Holm-adjusted p-values

## 8. Main Findings

- The mean `G3` score is approximately 10.42.
- `failures` and intention to pursue higher education (`higher`) show notable associations with
  `G3`.
- Regression without `G1/G2` has limited explanatory power; adding `G1/G2` improves the model
  substantially, but this remains predictive association rather than causal evidence.
- With `G3` standard deviation around 4.58, an experiment designed to detect a 1.5-point effect
  with about 80% power requires approximately 148 students per group, or 296 students in total.
